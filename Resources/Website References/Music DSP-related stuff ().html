Hi Darwin,
  Check out the diagram here: http://www.donreiman.com/Flanger/Flanger.htm

A flanger works like this:   y[n] = x[n] + wetness*x[n-delay] + feedback*y[n-1]

where delay = 0.5*sample_rate / ( center_frequency + mod_depth* LFO[phasor] )

(This will be a float so you need to make an interpolated delay line for sub-sample delays)

your LFO is a lookup table, so you will need a phasor (sawtooth oscillator between scaled between 0 and 1) to
read it.

It is also common to use a triangle wave LFO rather than a sine wave.

-David

The flanger is a kind of comb filter which delay *D* is not constant, but
changes periodically.
[image: flanger-schem]
Pic.8. The scheme of FIR flanger

The difference equation is *y(n)=x(n)+ax(n-D(n))*, where, for example
*D(n)=d/2(1-cos(2pFn))*,* F* - deviation speed (about 1 Hz), d - deviation
rate.
The modulation of delay time causes dynamic changes in dips and peaks
position. Additional effect is time stretching of audio signal. The time
delay varies between 1 and 5 ms. Alike echo there is *FIR* and* IIR*
version.

In the meantime, you can make a phasor and use the sine function:

float normalized_freq = lfo_freq/samplerate;
float phase = 0.f;

  // naive phasor oscillator, this part goes in your process loop
  phase += normalized_freq;
  if (phase >= 1.0) {

  }
  float lfo = sin( 2*M_PI * phase );

Re: LFO Flanger.

Ethan Duni <ethan.duni@...>
2014-06-27 04:28:39 GMT
Hi Darwin-

The thing is that D(n) is continuous-valued. So a delay line using an
integer number of samples (like x[n+D]) is problematic to get a good
flanger effect.

One option is to upsample the signal x[n] by a large factor for flanger
processing. Then you can use a simple integer delay line to good effect.
But this is inefficient.

Another option is to build a fractional delay unit, so you can estimate
values like x[n+1.5] directly. There are various trade-offs with these
filters, and making them for a widely variable delay value is also
challenging.

You can also use a combination of the two approaches. Upsampling the signal
moderately can ease the design constraints on the fractional delay
interpolator.

You should Google "fractional delay interpolation" or "fractional delay
filter" for some good references. Also look into Legendre polynomials.

E
